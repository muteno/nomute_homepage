노뮤트 홈페이지 UI/UX 개선 보고

에디터 디자인 기틀 · RSS 기능 이식 — 260702 · PR #1 (Cloudflare Pages 빌드 ✅)

한눈에. 노뮤트 에디터의 디자인 토큰·글래스 카드·스태거·편향 배지와 RSS 파이프라인을 홈페이지에 이식했습니다. 옛 60칸 수동 피드 나열을 자동 편성 엔진으로 대체하고, Opus 10인 적대 평의회로 검증해 발견을 전부 반영했습니다. 보안 CRITICAL(저장형 XSS) 1건을 포함해 접근성·대비·성능 결함을 함께 해결했습니다.

쉬운 설명 — 무엇이 바뀌었나

홈페이지는 그동안 각 언론사 뉴스를 손으로 60칸 나열했고, 색·버튼·간격도 에디터와 따로 놀았습니다. 이번에 ① 뉴스를 알아서 채우는 편성 엔진으로 바꾸고, ② 에디터에서 쓰던 디자인 값을 그대로 가져와 통일했으며, ③ 폰에서 탭·확대·키보드가 제대로 먹히도록 고치고, ④ 안 쓰는 4.9MB 영상·2.6MB 배경을 걷어내 훨씬 가볍게 만들었습니다.

전 / 후 비교

Before

  • 60칸 피드를 코드에 손으로 나열
  • 피드 죽으면 "기사를 불러올 수 없습니다" 빈 카드
  • 색·버튼이 에디터와 불일치(자체 팔레트)
  • 카드가 키보드·스크린리더로 열리지 않음
  • 더보기 버튼이 본문 텍스트를 가림(중앙 고정)
  • 시계가 접속 기기 시간(가짜 KST)
  • 미참조 영상 4.9MB + 배경 PNG 2.6MB

After

  • 소스 큐→중복제거→성향 슬롯 자동 편성
  • 죽은 피드는 폴백이 흡수 = 빈 카드 0
  • 에디터 :root 토큰 이식(색·반지름·모션 통일)
  • 스트레치드 링크 = 키보드·SR 진입 가능
  • 더보기·TOP 하단 우측 앵커+safe-area
  • KST 강제(Intl Asia/Seoul)·상대시간 실시간
  • 영상 삭제 + 배경 WebP 26KB

검증 — Opus 10인 적대 평의회

각기 다른 앵글(RSS 파싱·편성·클라JS·디자인토큰·모바일UX·성능·회귀·빌드·보안·총괄)로 병렬 감사한 뒤 발견을 반영하고 재빌드·Playwright 실측으로 확인했습니다.

품질 저하 리스크

  • 낮음 — 시각/기능 회귀. 카드·배지·스태거·더보기·TOP·헤더 전환을 Playwright로 실측 확인. 60개 스트레치드 링크·0 data-link·배지 4색 계산값 검증 완료.
  • 낮음 — 외부 이미지 실패율. 한국 언론 CDN이 핫링크를 막을 수 있어 referrerpolicy=no-referrer로 성공률을 높이고, 실패 시 카드가 무이미지 변형으로 우아하게 접힘.
  • 중간(구조적) — 정적 스냅샷. 아래 후속 참고.
