## 0. 한 줄 요지
디자인·UI 제안/시안/튜닝은 **말·정적 이미지로 하지 않는다** — 운영자가 **만져서 고르는 자기완결 HTML 1파일**(실물 재현 미리보기 + 프리셋 + 슬라이더 + 선택값 복사)로 준다. 운영자의 [선택값 복사] 답장이 **곧 구현 스펙**이 되어 전달 손실이 0이 된다. ← 결정 품질의 핵심 = *스샷 보고 상상*이 아니라 *실물 조건에서 조작·비교*.

## 1. (A) 개념규칙 — 대상 레포 `CLAUDE.md`에 붙일 블록 (플레이스홀더 `<…>`만 치환)

```markdown
## 🎛 디자인 제안 = 플레이그라운드식 HTML (항상)
디자인·UI 제안/시안/튜닝 = **플레이그라운드식 HTML이 디폴트**(정적 이미지·텍스트 나열로 갈음 금지). 산출 = `<보고서경로>/{yymmdd}_{라벨}.html` 자기완결 1파일(자산 base64 임베드 · 외부 요청 0 · 인라인 CSS/JS).
구성 5요소(전부 필수):
- ① 실물 재현 미리보기 — 실제 화면 프레임(<프레임>) 안에 `<SSOT>` 토큰 사본 + 정본 컴포넌트·모션 계승으로 재현. **현행 상태 = 비교 기준으로 함께 재현(고정)** · 튜닝 변수 기본값 = 현행 실측값.
- ② 프리셋 3~4개 — 현행 포함 · 추천 ★ 1개.
- ③ 미세조정 컨트롤 — 축별 그룹 라벨 · 값 표시 = tabular-nums · **옵션 규모 = 평균 50개 안팎**(항목 최대 고도화 — 단순 대상만 축소 허용) · **축 3분류(기틀 인지)**: {정렬·간격·높이 = 자유} · {글자·라운드 등 확립 축 = 자유 조절 + 실존값 사다리 눈금, 사다리 밖 = ⚠ 실시간} · {색 = 팔레트/토큰 폐쇄 셀렉트(자유 hex 금지) — 역할 재지정 = ⚠} · **복사 시 ⚠ 축 존재 = 알림창 + 「기틀 갱신 역제안」 블록 포함**(회신 = 기틀 갱신 승인 정문 경로 · 같은 토큰 알파 변주 = 자유).
- ④ 선택값 출력 + 복사 버튼 — 값마다 현행 병기 · 기존 토큰 동값 = "계승" / 아니면 "갱신 후보" 자동 표기 · 기틀 축 = ⚠ 딱지.
- ⑤ 재현 한계 각주 — 폴백·근사·실물과 다른 점 명시.
검증 = 헤드리스 브라우저 렌더 실측(상태 전환 실클릭 + 프리셋별 스크린샷 + JS 에러 0) 후에만 전달. 전달 = 레포 커밋(정본) + **채팅 파일 직접 첨부**(원탭) · Artifact 링크 = 같은 시안 반복 수정 루프·외부 공유 시 보조.
후속 = 운영자가 [선택값 복사] 결과를 붙이면 그 값을 `<SSOT>`·CSS에 배선(+`<게이트>` 통과)까지.
```

> 규칙: **얇게 유지** — 골격 코드·상세는 이 포터블 파일을 포인터로(본문 복제 = 드리프트).

### 2-갓차. 이식 갓차 3점
- **토큰이 없는 레포** — `<SSOT>`가 없으면 미리보기 재현은 그 레포 실물 CSS에서 값을 실측 발췌해 상수화(임의 창작 금지). 개념(현행 기준 고정·계승/갱신 표기)은 여전히 유효.
- **프레임은 실사용 화면비** — 모바일 앱을 데스크톱 폭으로 재현하면 결정이 틀어진다. 대상 사용자가 실제로 보는 프레임으로.
- **원격 자산 차단 대비** — 개발 환경이 CDN·버킷을 막을 수 있다. 받을 수 있으면 축소 후 base64 임베드, 못 받으면 실물의 폴백 경로(이니셜·플레이스홀더)로 재현하고 ⑤ 각주에 명시.

## 3. 골격 스펙 — 제작 패턴 (핵심 코드 그대로 · 전문은 `<확립본>`)

### 3-1. 레이아웃 뼈대
```html
<div class="wrap"><!-- flex-wrap: 좌 미리보기 · 우 콘솔(폰에선 세로 쌓임) -->
```

### 3-2. 튜닝 변수 규약 ★핵심★
```css
.app{
  /* --t-* 프리픽스 · 미리보기 루트 스코프 · ⚠ 기본값 = 반드시 현행 실측값
     (기본값이 추천값이면 "전/후" 비교의 기준이 사라져 결정이 왜곡된다) */
  --t-dim:.66; --t-glass:.055; /* … 축마다 하나 */
}
/* 실물 CSS를 복사하되 밝기·크기 등 튜닝 축만 var(--t-*)로 치환 */
.card{ background:rgba(255,255,255,var(--t-glass)); }
```

### 3-2′. 축 3분류 — 기틀 인지 + 이탈 시 「기틀 갱신 역제안」 (옵션 설계 규약 ★)
옵션은 **평균 50개 안팎**(복잡한 카드·차트·화면은 70개+)으로 풍성하게, **가리지 않는다** — 확립 축(글자·라운드 등)도 자유 조절을 막지 않는다(운영자가 기틀 자체를 바꾸고 싶을 수 있다). 대신 기틀 변경이 *몰래*가 아니라 **명시적 역제안**이 되게 3중 장치를 단다:
- **① 확립값 안내**: 확립 축은 자유 슬라이더에 **그 레포 실존값 사다리를 실측 추출**(예: `font-size` 정규식 빈도 집계)해 눈금(datalist)으로 얹고, 사다리 안 = `계승` 판정 표기.
- **② 실시간 이탈 표시**: 값이 사다리·토큰 밖으로 나가거나 역할색을 재지정하면 그 행에 **⚠ 즉시 표시** + 콘솔에 「기틀 이탈 N축」 카운터.
- **③ 복사 시 알림창 + 역제안**: [선택값 복사]에 ⚠ 축이 있으면 **알림창(모달)** — {이탈 축 목록 · 이대로 복사하면 「🔁 기틀 갱신 역제안」 블록 포함 · 회신 = 그 값으로 **기틀 자체(전 앱 동일 축 일괄)를 바꾸는** 그 레포 승인 정문 경로} 고지 후 [역제안 포함 복사] / [계승분만 복사] / [취소] 선택. 출력의 역제안 블록엔 축별 {현행 기틀값 → 제안값 · 적용 범위 = 전 앱 동축}을 명시.
- **색만은 팔레트/토큰 폐쇄 셀렉트 유지**(자유 hex 피커 금지 — 즉석 창작은 팔레트 폐쇄 원칙 위반. 팔레트 **안** 재지정은 전 칩 개방 + ⚠ 역제안 대상. 새 hue 필요 = 그 레포 색 정문으로).
- 같은 토큰의 알파 변주(rgba 알파)는 자유 슬라이더 허용(디자인 계약상 신규색 아님).
- 프리셋도 동일 규율: 추천(★) 프리셋은 지오메트리 위주로, 역할색·이탈 값을 쓰는 프리셋은 이름에 ⚠.

### 3-3. 상태 ⇄ 슬라이더 ⇄ 출력 바인딩 (JS 패턴)
```js
const DEFAULTS = {dim:.66, glass:.055 /* 현행 실측 */};
const PRESETS = [
  {id:'p0', nm:'현행', st:{...DEFAULTS}},
  {id:'p1', nm:'A …', st:{...DEFAULTS, dim:.34}},
  {id:'p2', nm:'B …', star:'★', st:{...DEFAULTS, dim:.22, glass:.09}},
];
const SLIDERS = [ {k:'dim', nm:'라벨', d:'대상 셀렉터/토큰', min:0, max:.8, step:.01, grp:'그룹', warn:0} /* warn:1 = 기틀 ⚠ */ ];
let state = {...DEFAULTS};
function apply(){                       // 단일 반영점: state → CSS 변수 + 슬라이더 UI + 출력
  for (const k in state) app.style.setProperty('--t-'+k, state[k]);
  /* 슬라이더 value·트랙 채움(--p)·값 라벨(tabular-nums) 동기 */ buildOut();
}
/* 슬라이더 input → state[k] 갱신 → markCustom()(프리셋 해제 표시) → apply()
   프리셋 클릭 → state = {...p.st} → apply() */
```

### 3-4. 선택값 출력 — 현행 병기 + 계승/갱신 자동 표기 ★전달 손실 0의 지점★
```js
const near = (a,b,eps=.004) => Math.abs(a-b) < eps;   // 기존 토큰 동값 판정
function buildOut(){
  const note = near(state.glass,.09) ? ' = --glass-2 동값(계승)' : ' → 갱신 후보';
  out.textContent =
`[<라벨> — 선택값]
딤   : rgba(0,0,0,${state.dim})   [현행 .66]
표면 : rgba(255,255,255,${state.glass})${note}
배경 : ${state.lift ? '… ⚠기틀 갱신' : '현행 유지(기틀)'}`;
}
// 복사: navigator.clipboard → 실패 시 Range 선택 + execCommand 폴백(구형 웹뷰)
```

### 3-5. 재현 원칙
- 미리보기 = **그 레포 토큰·정본 컴포넌트·모션 커브를 그대로 계승**(시안 자체가 디자인 계약을 위반하면 안 됨). 임의 창작 재현 금지.
- 자산 = base64 임베드(자기완결·전달 통로 무관 동일 렌더). 큰 이미지는 표시 크기 2x로 축소 후 임베드.
- 상태 전환(잠금→언락 같은 흐름)이 주제면 **전환 연출까지 실물 커브로 재현** — 전환 자체가 결정 대상인 경우가 많다.

## 4. 검증 (필수) — "떠서 보임"이 아니라 "만져짐"까지
1. 헤드리스 브라우저(Playwright 등)로 파일 열기 → `pageerror`·콘솔 에러 **0**.
2. 상태 전환·프리셋을 **실클릭**으로 구동(무한 애니 요소는 force 클릭) → 상태별 스크린샷을 직접 눈으로 확인.
3. 통과 후에만 커밋·전달. (yeta 실측 선례: 셀렉터 오타로 전환 불능이던 것을 이 절차가 잡음 — 안 만져봤으면 깨진 채 전달됐다.)

## 5. 전달 통로 판정 — HTML 첨부 vs Artifact (비용·효율·품질)
- **품질(최우선)**: base64 임베드 원칙을 지키면 **어느 통로든 렌더 동일 = 결정 품질 동률**. 실패 모드만 다름 — Artifact는 외부 자산 원천 차단(임베드 누락 시 폴백으로 보게 됨), 첨부는 기기 뷰어 의존. 판정 기준은 "실물과 같은 조건으로 만져지는가" 하나.
- **비용**: 차이 무의미(양쪽 다 동일 파일 · Artifact는 게시 1단계 추가).
- **효율**: 1회성 시안 = **첨부가 빠름**. 같은 시안을 여러 번 고치는 튜닝 루프 = **Artifact 병행이 유리**(같은 URL 재배포 = 링크 하나로 최신판).
- **규칙**: 기본 = 레포 커밋(정본) + 채팅 파일 직접 첨부 · 반복 수정 루프 진입 or 외부 공유 요청 시 Artifact 병행.

## 6. 하지 말 것 (안티패턴 — AI가 흔히 망치는 지점)
- ✗ PNG·스샷 나열로 갈음(만질 수 없음 = 결정 품질 하락 — 스샷은 채팅 보조로만).
- ✗ 튜닝 변수 기본값을 추천값으로(기본 = **현행 실측** — 아니면 전/후 비교가 거짓).
- ✗ 현행 비교 기준 생략(프리셋에 "현행"이 없으면 정도를 잴 자가 없음).
- ✗ 외부 URL 자산 참조(환경 따라 깨짐 — base64 임베드).
- ✗ 선택값 출력 생략("보고 말로 정해줘" = 전달 손실·재질문 루프).
- ✗ 미검증 전달(헤드리스 실클릭 + 스크린샷 + JS 에러 0 전엔 미완성).
- ✗ 그 레포 디자인 계약을 어긴 재현(토큰·정본 컴포넌트 계승 — 시안도 계약 안이다).
- ✗ 확립 축을 **이탈 감지 없이** 자유 슬라이더로(조용한 기틀 파괴 — §3-2′ 사다리 눈금 + ⚠ 실시간 + 복사 알림창·역제안 필수). 반대로 **아예 잠그는 것도 금지**(운영자의 기틀 갱신 의지를 가림 — 막지 말고 역제안으로 승격).
- ✗ 옵션을 몇 개만 주고 끝(§3-2′ 평균 50개 안팎이 기본 — 적으면 사용자가 고도화 설계를 못 한다).
